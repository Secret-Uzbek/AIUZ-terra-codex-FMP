# Contributing to AIUZ Terra Codex FMP

This repository accepts contributions for the FMP theory and publication layer.

Before contributing, read:

- `README.md`
- `DOCUMENTATION-STANDARD.md`
- `SECURITY.md`
- `CODE_OF_CONDUCT.md`
- the donor governance and glossary files in `terra-legal`

## What belongs here

Appropriate contributions include:

- article cleanup and publication repair;
- bibliography integration and citation repair;
- documentation standardization;
- living-index repair;
- controlled translation or language cleanup;
- detox of encoding damage, duplicates, and false service files.

## What does not belong here

Do not use this repository as a general drop zone for every Terra artifact.

Move or archive instead of expanding entropy when the file is:

- unrelated to FMP;
- a duplicate of a stronger file;
- a broken export or temporary residue;
- a purely local machine artifact.

## Working sequence

Use this order:

1. Read the target file and its neighbors.
2. Check whether a stronger version already exists.
3. Repair encoding, structure, and provenance before adding new text.
4. Keep English as the main public surface unless the file is explicitly multilingual.
5. Preserve trace value and source clarity.

## Pull request expectations

- Keep changes focused and explainable.
- Do not mix unrelated detox, publication, and theory changes in one pass.
- Prefer improving an existing file over creating a new parallel copy.
- If you touch standards, security, governance, or citation files, keep them aligned with `terra-legal`.

## Publication discipline

If a contribution affects an article, white paper, or public artifact:

- keep metadata coherent;
- preserve or improve citations;
- do not remove evidence without replacement;
- keep repository DOI and release logic intact.

## Security and disclosure

If you find a workflow, token, credential, publication, or privacy problem, use
the reporting route in `SECURITY.md` instead of opening a public exploit trail.
