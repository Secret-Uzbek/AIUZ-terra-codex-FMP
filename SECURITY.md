# Security Policy

## Scope

This repository is a publication-facing and documentation-facing FMP surface.

Security work here mainly concerns:

- accidental publication of credentials or tokens;
- broken or unsafe release workflows;
- exposed personal data;
- tampering with citation, DOI, or publication metadata;
- malicious or misleading file replacement inside the repository.

## Supported branch

The actively maintained branch is:

- `main`

## How to report

Do not open a public issue for a security-sensitive problem.

Report privately to:

- `a.abdukarimov@fractal-metascience.org`

If that channel is unavailable, use the repository owner contact routes already
published across the Terra ecosystem.

## What to include

Please include:

- the affected file or workflow;
- a short description of the problem;
- steps to reproduce if relevant;
- whether the issue affects publication integrity, personal data, or release automation.

## What not to do

Do not publish secrets, tokens, exploit steps, or sensitive traces in public
threads.
